> **模板名称**: 测试文件模板
> **版本**: v1.0
> **分类**: A-模块开发
> **层级**: 精简版

---

## 模板说明

测试文件采用"分组-用例-断言"的三层结构，确保测试清晰、可维护。
以下为通用的最小化测试文件模板。

---

## 基本测试结构

```
describe("｛｛MODULE_NAME｝｝", () => {

  describe("｛｛FUNCTION_GROUP｝｝", () => {

    test("｛｛TEST_CASE_DESCRIPTION｝｝", () => {
      // 准备测试数据
      const input = ｛｛INPUT_DATA｝｝;

      // 执行被测函数
      const result = ｛｛FUNCTION_CALL｝｝;

      // 断言结果
      assert.equal(result, ｛｛EXPECTED_VALUE｝｝);
    });

  });

});
```

---

## 结构说明

| 层级 | 关键字 | 说明 |
|------|--------|------|
| 分组 | `describe` | 按模块/功能分组，可嵌套 |
| 用例 | `test` | 单个测试用例，描述具体场景 |
| 断言 | `assert` | 验证实际结果是否符合预期 |

---

## 使用示例

假设测试一个字符串反转函数：

```
describe("string-utils", () => {

  describe("reverse", () => {

    test("应正确反转普通字符串", () => {
      const input = "hello";
      const result = reverse(input);
      assert.equal(result, "olleh");
    });

    test("应正确处理空字符串", () => {
      const result = reverse("");
      assert.equal(result, "");
    });

  });

});
```
