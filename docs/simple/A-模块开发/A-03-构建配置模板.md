## 模板说明

构建配置文件定义了模块如何从源代码编译为可分发的产物。
以下为通用的最小化构建配置模板。

---

## 最小化构建配置

```json
{
  "entry": {
    "index": "｛｛入口文件｝｝"
  },
  "output": {
    "formats": ["esm", "cjs"],
    "dir": "｛｛输出目录｝｝"
  },
  "sourcemap": true,
  "clean": true
}
```

---

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `entry` | 是 | 入口文件映射，键为产物名称，值为源文件路径 |
| `output.formats` | 是 | 输出模块格式：`esm`（ES 模块）、`cjs`（CommonJS） |
| `output.dir` | 是 | 产物输出目录 |
| `sourcemap` | 否 | 是否生成源码映射文件，调试时建议开启 |
| `clean` | 否 | 构建前是否清除输出目录，建议开启 |

---

## 使用示例

```json
{
  "entry": {
    "index": "src/index"
  },
  "output": {
    "formats": ["esm", "cjs"],
    "dir": "dist"
  },
  "sourcemap": true,
  "clean": true
}
```

构建后产物结构：

```
dist/
├── index.esm       # ES 模块格式
├── index.cjs       # CommonJS 格式
└── index.esm.map   # 源码映射
```
