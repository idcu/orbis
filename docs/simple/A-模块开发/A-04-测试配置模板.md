# A-S04 测试配置模板（精简版）

> **模板编号**: A-S04
> **模板名称**: 测试配置模板
> **版本**: v1.0
> **分类**: A-模块开发
> **层级**: 精简版

---

## 模板说明

测试配置文件定义了测试框架的运行参数，包括测试文件发现规则和输出格式。
以下为通用的最小化测试配置模板。

---

## 最小化测试配置

```json
{
  "testDirectory": "｛｛测试目录｝｝",
  "testFilePattern": "｛｛测试文件匹配模式｝｝",
  "reporter": "｛｛报告格式｝｝"
}
```

---

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `testDirectory` | 否 | 测试文件所在目录，默认为 `tests/` 或 `__tests__/` |
| `testFilePattern` | 否 | 测试文件匹配模式，如 `*.test.*` 或 `*.spec.*` |
| `reporter` | 否 | 测试结果输出格式，如 `verbose`、`dot`、`json` |

---

## 使用示例

```json
{
  "testDirectory": "tests",
  "testFilePattern": "*.test.*",
  "reporter": "verbose"
}
```

对应的测试目录结构：

```
tests/
├── core.test.*        # 核心功能测试
└── utils.test.*       # 工具函数测试
```
