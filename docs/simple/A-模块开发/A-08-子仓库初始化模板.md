# A-S08 子仓库初始化模板（精简版）

> **模板编号**: A-S08
> **模板名称**: 子仓库初始化模板
> **版本**: v1.0
> **分类**: A-模块开发
> **层级**: 精简版

---

## 模板说明

在多包仓库（monorepo）中，每个子模块需要遵循统一的目录结构和配置规范。
以下为精简版的子仓库初始化模板。

---

## 目录结构

```
｛｛模块名称｝｝/
├── src/
│   └── index          # 模块入口文件
├── tests/
│   └── index.test     # 测试文件
├── ｛｛清单文件｝｝  # 模块清单文件
└── README.md          # 模块说明文档
```

---

## 模块清单文件

```json
{
  "name": "｛｛模块名称｝｝",
  "version": "0.1.0",
  "description": "｛｛描述｝｝",
  "main": "src/index",
  "scripts": {
    "build": "｛｛构建命令｝｝",
    "test": "｛｛测试命令｝｝",
    "lint": "｛｛代码检查命令｝｝"
  },
  "license": "｛｛许可证｝｝"
}
```

---

## 仓库级工作区配置

在仓库根目录的工作区配置文件中注册新模块：

```json
{
  "packages": [
    "packages/｛｛模块名称｝｝",
    "｛｛已有包｝｝"
  ]
}
```

---

## 初始化步骤

1. 在 `packages/` 目录下创建模块目录
2. 按照上述目录结构创建文件
3. 填写模块清单文件
4. 在仓库根目录的工作区配置中注册模块
5. 在仓库根目录执行依赖安装命令
6. 验证构建和测试通过
