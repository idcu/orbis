> **版本**: v1.0
> **分类**: A-模块开发
> **层级**: 精简版

---

## 模板说明

模块清单文件是每个模块的元数据描述文件，用于声明模块的基本信息、入口、依赖和脚本命令。
不同生态系统中清单文件的名称不同，但核心字段相通：

| 生态系统 | 清单文件名 |
|----------|-----------|
| 通用概念 | 模块清单文件 |
| 示例 A | `package.json` |
| 示例 B | `Cargo.toml` |
| 示例 C | `go.mod` |
| 示例 D | `build.gradle` / `pom.xml` |

---

## 最小化清单模板

```json
{
  "name": "｛｛MODULE_NAME｝｝",
  "version": "｛｛VERSION｝｝",
  "description": "｛｛DESCRIPTION｝｝",
  "main": "｛｛ENTRY_FILE｝｝",
  "scripts": {
    "build": "｛｛BUILD_COMMAND｝｝",
    "test": "｛｛TEST_COMMAND｝｝",
    "lint": "｛｛LINT_COMMAND｝｝"
  },
  "dependencies": {
    "｛｛INTERNAL_DEP_NAME｝｝": "｛｛WORKSPACE_PROTOCOL｝｝:｛｛WORKSPACE_PATH｝｝"
  },
  "devDependencies": {
    "｛｛BUILD_TOOL｝｝": "｛｛BUILD_TOOL_VERSION｝｝",
    "｛｛TEST_FRAMEWORK｝｝": "｛｛TEST_FRAMEWORK_VERSION｝｝"
  },
  "license": "｛｛LICENSE｝｝"
}
```

---

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | 是 | 模块名称，全局唯一，小写+连字符 |
| `version` | 是 | 模块版本号，遵循语义化版本规范 |
| `description` | 是 | 一句话描述模块用途 |
| `main` | 是 | 模块主入口文件路径 |
| `scripts` | 是 | 常用脚本命令映射 |
| `dependencies` | 否 | 运行时依赖列表 |
| `devDependencies` | 否 | 开发时依赖列表 |
| `license` | 是 | 许可证标识 |

---

## 使用示例

假设创建一个名为 `string-utils` 的工具模块：

```json
{
  "name": "@myorg/string-utils",
  "version": "0.1.0",
  "description": "通用字符串处理工具集",
  "main": "src/index",
  "scripts": {
    "build": "build-tool --output dist",
    "test": "test-runner",
    "lint": "linter src/"
  },
  "dependencies": {},
  "devDependencies": {
    "build-tool": "^2.0.0",
    "test-runner": "^1.0.0"
  },
  "license": "MIT"
}
```
