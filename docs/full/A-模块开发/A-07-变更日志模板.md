## 模板说明

变更日志（CHANGELOG）用于记录模块每个版本的所有重要变更。
本模板基于 [Keep a Changelog](https://keepachangelog.com/) 规范，
版本号遵循 [语义化版本](https://semver.org/) 规范。

---

## 变更分类说明

| 分类 | 英文 | 适用场景 | 版本影响 |
|------|------|----------|----------|
| 新增 | Added | 新增功能、特性 | 次版本号（MINOR）或修订号（PATCH） |
| 变更 | Changed | 对现有功能的修改 | 视兼容性而定 |
| 废弃 | Deprecated | 即将在未来版本移除的功能 | 通常为次版本号（MINOR） |
| 移除 | Removed | 本版本移除的功能 | 主版本号（MAJOR） |
| 修复 | Fixed | 缺陷修复 | 修订号（PATCH） |
| 安全 | Security | 安全漏洞修复 | 视严重程度而定 |

## 编写规范

### 每条变更的格式

```
- 简洁描述变更内容
```

**良好示例**：

```markdown
### 新增
- 新增 `trim` 函数，支持去除首尾空白字符
- 新增 `padStart` 函数，支持字符串前缀填充

### 修复
- 修复 `capitalize` 在空字符串输入时的异常崩溃
- 修复 `reverse` 处理多字节字符时结果不正确的问题
```

**不良示例**：

```markdown
### 新增
- 加了一些新功能
- 修复了 bug
```

### 版本比较链接

每个版本应包含与前一版本的比较链接，便于查看具体代码变更：

```markdown
## [1.2.0] - 2026-04-15

[1.2.0]: https://｛｛主机名｝｝/｛｛组织名｝｝/｛｛仓库名｝｝/compare/v1.1.0...v1.2.0
```

### 废弃说明

废弃某个功能时，应说明替代方案和预计移除时间：

```markdown
### 废弃
- 废弃 `oldFunction`，请迁移至 `newFunction`，将在 2.0.0 中移除
```

## 使用示例

```markdown
# 变更日志

本项目的所有重要变更均记录在此文件中。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/)，
版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [0.2.0] - 2026-04-15

0.2.0 相对于 [0.1.0](https://example.com/org/repo/compare/v0.1.0...v0.2.0) 的变更：

### 新增
- 新增 `trim` 函数，支持去除首尾空白字符
- 新增 `padStart` 和 `padEnd` 函数，支持字符串填充
- 新增 `split` 函数，支持按分隔符拆分字符串

### 变更
- `reverse` 函数性能优化，处理长字符串时提升约 30%
- 统一所有函数的参数校验错误信息格式

### 废弃
- 废弃 `oldTrim`，请迁移至 `trim`，将在 1.0.0 中移除

### 修复
- 修复 `capitalize` 在空字符串输入时的异常崩溃
- 修复 `reverse` 处理多字节字符时结果不正确的问题
- 修复 `repeat` 在负数参数时的未定义行为

## [0.1.0] - 2026-03-01

### 新增
- 初始发布
- 包含 `reverse`、`capitalize`、`repeat` 三个基础函数
```
