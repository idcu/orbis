# A-F01 新模块创建检查清单（完整版）

> **模板 ID**: A-F01
> **模板名称**: 新模块创建检查清单（完整版）
> **版本**: 1.0.0
> **适用范围**: 新建模块/包时的全面自检，确保所有配置和规范到位

---

## 检查清单

在创建新模块时，逐项确认以下 9 个类别中的所有检查项。

---

### 一、模块清单文件字段完整性

- [ ] **name**：模块名称，格式为 `@｛｛作用域｝｝/｛｛模块名称｝｝` 或 `｛｛模块名称｝｝`
- [ ] **version**：初始版本号 `0.1.0`
- [ ] **description**：一句话描述模块用途
- [ ] **main**：主入口文件路径（如 `src/index`）
- [ ] **module**（如适用）：ESM 入口文件路径
- [ ] **types**（如适用）：类型声明文件入口路径
- [ ] **exports**（如适用）：条件导出映射
- [ ] **files**（如适用）：发布时包含的文件/目录白名单
- [ ] **sideEffects**：标记是否具有副作用（影响 tree-shaking）
- [ ] **license**：许可证标识，与仓库主许可证一致
- [ ] **repository**：仓库地址
- [ ] **keywords**：便于搜索的关键词列表
- [ ] **author** / **contributors**：作者与贡献者信息

---

### 二、类型/接口定义

- [ ] **接口定义文件**：在 `src/` 下创建类型/接口定义文件
- [ ] **公开类型导出**：所有公开接口的类型从入口文件导出
- [ ] **内部类型隔离**：仅内部使用的类型不对外导出
- [ ] **文档注释**：每个公开接口附带描述性注释
- [ ] **参数与返回值类型**：所有函数参数和返回值均有明确类型标注

---

### 三、构建配置

- [ ] **入口配置**：正确指定入口文件路径
- [ ] **输出格式**：根据目标环境配置 ESM / CJS 输出
- [ ] **类型声明生成**：启用声明文件自动生成
- [ ] **源码映射**：开发阶段启用 source map
- [ ] **目标环境**：指定编译目标（如现代浏览器 / 服务端 / 通用）
- [ ] **外部依赖**：将运行时依赖标记为外部，不打包进产物
- [ ] **Tree-shaking**：确保产物支持 tree-shaking（ESM + 无副作用标记）
- [ ] **压缩**：生产构建启用代码压缩
- [ ] **清理**：构建前自动清除旧产物目录

---

### 四、测试配置

- [ ] **测试目录**：指定测试文件目录（如 `tests/`）
- [ ] **文件匹配模式**：配置测试文件发现规则（如 `*.test.*`）
- [ ] **覆盖率阈值**：设置最低覆盖率要求（建议 >= 80%）
- [ ] **覆盖率报告格式**：配置报告输出格式（文本 / HTML / JSON）
- [ ] **全局设置**（如适用）：注入测试所需的公共工具函数
- [ ] **超时时间**：为慢速测试设置合理的超时上限
- [ ] **并行执行**：启用测试并行运行以提升速度

---

### 五、文档

- [ ] **README.md**：包含模块名称、描述、安装、用法、许可证
- [ ] **API 文档**：公开接口的详细说明（参数、返回值、示例）
- [ ] **CHANGELOG.md**：变更日志文件，记录版本变更
- [ ] **代码内注释**：复杂逻辑处添加行内注释

---

### 六、发布配置

- [ ] **发布内容白名单**：通过 `files` 字段或 `.npmignore` / `.publishignore` 控制发布内容
- [ ] **忽略列表**：排除测试文件、源码映射、配置文件、构建脚本等
- [ ] **发布脚本**（如适用）：配置自动化发布流程
- [ ] **版本管理**：遵循语义化版本规范
- [ ] **变更日志更新**：每次发布前更新 CHANGELOG

---

### 七、仓库级集成

- [ ] **工作区注册**：在仓库根目录的工作区配置中注册新模块
- [ ] **依赖安装验证**：在仓库根目录执行依赖安装，确认模块被正确识别
- [ ] **构建集成**：仓库级构建命令能正确构建该模块
- [ ] **测试集成**：仓库级测试命令能正确运行该模块的测试
- [ ] **代码检查集成**：仓库级代码检查命令覆盖该模块
- [ ] **文档生成集成**（如适用）：API 文档生成工具能正确处理该模块

---

### 八、代码规范

- [ ] **代码检查工具配置**：继承仓库级配置或创建模块级配置
- [ ] **代码格式化工具配置**：与仓库保持一致的格式化规则
- [ ] **提交规范**：提交信息遵循约定式提交（Conventional Commits）规范
- [ ] **编辑器配置**：`.editorconfig` 确保编辑器设置一致（缩进、字符集等）
- [ ] **忽略文件**：`.gitignore` 排除构建产物、依赖目录、临时文件等

---

### 九、安全检查

- [ ] **依赖安全审计**：所有依赖无已知高危漏洞
- [ ] **无敏感信息**：代码中无硬编码的密钥、Token、密码等
- [ ] **输入校验**：对外部输入进行了充分的校验和清理

---
