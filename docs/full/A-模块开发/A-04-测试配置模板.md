# A-F04 测试配置模板（完整版）

> **模板 ID**: A-F04
> **模板名称**: 测试配置模板（完整版）
> **版本**: 1.0.0
> **适用范围**: 模块/包的完整测试框架配置

---

## 模板说明

测试配置文件定义了测试框架的全部运行参数，包括文件发现、覆盖率、全局设置、超时等。
以下为通用的完整测试配置模板，可映射到任意测试框架。

---

## 通用字段参考

| 字段 | 必填 | 说明 |
|------|------|------|
| `testDirectory` | 否 | 测试文件目录 |
| `testFilePattern` | 否 | 测试文件匹配模式 |
| `reporter` | 否 | 测试结果输出格式 |
| `coverage.threshold` | 否 | 覆盖率阈值 |
| `coverage.reporters` | 否 | 覆盖率报告格式 |
| `coverage.include` | 否 | 覆盖率统计包含的源码路径 |
| `coverage.exclude` | 否 | 覆盖率统计排除的路径 |
| `globals` | 否 | 测试环境全局变量 |
| `setupFiles` | 否 | 测试前执行的初始化脚本 |
| `teardownFiles` | 否 | 测试后执行的清理脚本 |
| `pathAliases` | 否 | 路径别名映射 |
| `timeout` | 否 | 单个测试用例超时时间（毫秒） |
| `parallel` | 否 | 是否并行执行测试 |
| `maxWorkers` | 否 | 最大并行工作线程数 |

---

## 完整测试配置

```json
{
  "testDirectory": "tests",
  "testFilePattern": ["*.test.*", "*.spec.*"],
  "reporter": "verbose",

  "coverage": {
    "threshold": {
      "branches": 80,
      "functions": 80,
      "lines": 80,
      "statements": 80
    },
    "reporters": ["text", "html", "json-summary"],
    "include": ["src/**/*"],
    "exclude": [
      "src/**/*.d.*",
      "src/**/*.test.*",
      "src/**/*.spec.*",
      "src/**/index.*"
    ]
  },

  "globals": {
    "｛｛全局辅助函数名｝｝": true
  },

  "setupFiles": ["tests/setup.*"],
  "teardownFiles": ["tests/teardown.*"],

  "pathAliases": {
    "@｛｛作用域｝｝/shared": "｛｛共享模块工作区路径｝｝",
    "@｛｛作用域｝｝/test-utils": "｛｛测试工具工作区路径｝｝"
  },

  "timeout": 10000,
  "parallel": true,
  "maxWorkers": ｛｛CPU核心数｝｝
}
```

---

## 字段详解

### 覆盖率配置

| 子字段 | 说明 | 建议值 |
|--------|------|--------|
| `threshold.branches` | 分支覆盖率阈值 | >= 80% |
| `threshold.functions` | 函数覆盖率阈值 | >= 80% |
| `threshold.lines` | 行覆盖率阈值 | >= 80% |
| `threshold.statements` | 语句覆盖率阈值 | >= 80% |

**覆盖率报告格式**：

| 格式 | 说明 |
|------|------|
| `text` | 终端文本输出，适合 CI 日志 |
| `html` | HTML 报告，适合本地查看详细覆盖情况 |
| `json-summary` | JSON 摘要，适合 CI 门禁集成 |
| `lcov` | LCOV 格式，适合与第三方覆盖率服务集成 |

### 全局变量

通过 `globals` 配置可在测试文件中直接使用工具函数，无需手动导入：

```json
{
  "globals": {
    "assert": true,
    "mockFn": true
  }
}
```

### 初始化与清理

- **setupFiles**：在所有测试运行之前执行，适合初始化测试环境（如设置环境变量、创建临时目录）
- **teardownFiles**：在所有测试运行之后执行，适合清理测试环境（如删除临时文件、关闭连接）

### 路径别名

将模块路径别名映射到实际路径，确保测试中导入路径正确解析：

```json
{
  "pathAliases": {
    "@myorg/shared": "../shared/src",
    "@myorg/test-utils": "../test-utils/src"
  }
}
```

### 超时与并行

| 配置项 | 说明 | 建议值 |
|--------|------|--------|
| `timeout` | 单个测试用例最大执行时间 | 10000ms（10秒） |
| `parallel` | 是否启用并行执行 | `true` |
| `maxWorkers` | 最大并行线程数 | CPU 核心数或 `50%` |

## 测试目录结构

```
tests/
├── setup.*            # 测试初始化脚本
├── teardown.*         # 测试清理脚本
├── unit/              # 单元测试
│   ├── core.test.*
│   └── utils.test.*
├── integration/       # 集成测试（如适用）
│   └── workflow.test.*
└── fixtures/          # 测试固件/数据
    └── sample-data.*
```

## 配置最佳实践

| 实践 | 说明 |
|------|------|
| 覆盖率阈值 >= 80% | 确保基本代码质量底线 |
| 排除声明文件和入口文件 | 这些文件通常不包含可测逻辑 |
| 使用 setup/teardown | 避免每个测试文件重复初始化/清理代码 |
| 配置路径别名 | 与构建配置保持一致，避免导入路径问题 |
| 合理设置超时 | 避免慢速测试阻塞整个测试套件 |
| 启用并行执行 | 显著提升测试运行速度 |
