# A-F08 子仓库初始化模板（完整版）

> **模板 ID**: A-F08
> **模板名称**: 子仓库初始化模板（完整版）
> **版本**: 1.0.0
> **适用范围**: 在多包仓库中完整初始化一个新的子模块/子仓库

---

## 模板说明

在多包仓库（monorepo）中，每个子模块需要遵循统一的目录结构和配置规范。
完整版模板包含子仓库所需的全部文件和配置。

---

## 一、目录结构

```
｛｛模块名称｝｝/
├── src/
│   ├── index            # 模块入口文件
│   ├── types            # 类型/接口定义
│   └── utils            # 内部工具函数（如有）
├── tests/
│   ├── setup            # 测试初始化脚本
│   ├── teardown         # 测试清理脚本
│   └── index.test       # 测试文件
├── ｛｛清单文件｝｝    # 模块清单文件
├── ｛｛构建配置文件｝｝    # 构建配置文件
├── ｛｛测试配置文件｝｝ # 测试配置文件
├── README.md            # 模块说明文档
├── CHANGELOG.md         # 变更日志
├── ｛｛忽略文件｝｝   # 忽略文件配置
└── ｛｛代码检查配置文件｝｝ # 代码检查配置（如需覆盖）
```

## 三、仓库级工作区配置

在仓库根目录的工作区配置文件中注册新模块：

```json
{
  "packages": [
    "packages/｛｛模块名称｝｝",
    "packages/shared",
    "packages/cli",
    "｛｛其他已有包｝｝"
  ]
}
```


## 八、初始化步骤清单

### 步骤 1：创建目录结构

```bash
mkdir -p packages/｛｛模块名称｝｝/{src,tests}
```

### 步骤 2：创建基础文件

按照上述模板创建以下文件：
- `src/index` — 模块入口
- `src/types` — 类型定义
- `tests/index.test` — 测试文件
- `｛｛清单文件｝｝` — 模块清单
- `｛｛构建配置文件｝｝` — 构建配置
- `｛｛测试配置文件｝｝` — 测试配置
- `README.md` — 说明文档
- `CHANGELOG.md` — 变更日志
- `.gitignore` — 忽略配置

### 步骤 3：注册到工作区

在仓库根目录的工作区配置中添加模块路径。

### 步骤 4：安装依赖

```bash
# 在仓库根目录执行
｛｛工作区安装命令｝｝
```

### 步骤 5：验证

```bash
# 验证模块被正确识别
｛｛工作区列表命令｝｝

# 验证构建
｛｛工作区构建命令｝｝ --filter=｛｛模块名称｝｝

# 验证测试
｛｛工作区测试命令｝｝ --filter=｛｛模块名称｝｝

# 验证代码检查
｛｛工作区代码检查命令｝｝ --filter=｛｛模块名称｝｝
```

### 步骤 6：提交

```bash
git add packages/｛｛模块名称｝｝
git commit -m "feat: 初始化 ｛｛模块名称｝｝ 模块"
```
