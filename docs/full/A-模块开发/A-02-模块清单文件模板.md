# A-F02 模块清单文件模板（完整版）

> **模板 ID**: A-F02
> **模板名称**: 模块清单文件模板（完整版）
> **版本**: 1.0.0
> **适用范围**: 模块/包的清单文件完整配置，覆盖多种模块类型

---

## 模板说明

模块清单文件是每个模块的核心元数据文件，声明了模块的基本信息、入口、依赖、脚本命令和发布配置。

不同生态系统使用不同的清单文件格式，但核心概念相通。本模板使用通用的 JSON-like 格式，
可映射到任意生态系统：

| 生态系统 | 清单文件名 | 格式 |
|----------|-----------|------|
| 生态 A | `package.json` | JSON |
| 生态 B | `Cargo.toml` | TOML |
| 生态 C | `go.mod` | 自定义 |
| 生态 D | `build.gradle` / `pom.xml` | Groovy / XML |

---

## 通用字段参考

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | 是 | 模块名称 |
| `version` | 是 | 版本号（语义化版本） |
| `description` | 是 | 模块描述 |
| `main` | 是 | 主入口文件 |
| `module` | 否 | ESM 入口文件 |
| `types` | 否 | 类型声明入口 |
| `exports` | 否 | 条件导出映射 |
| `files` | 否 | 发布白名单 |
| `bin` | 否 | 可执行文件入口 |
| `scripts` | 是 | 脚本命令 |
| `dependencies` | 否 | 运行时依赖 |
| `devDependencies` | 否 | 开发依赖 |
| `peerDependencies` | 否 | 对等依赖（宿主环境提供） |
| `sideEffects` | 否 | 副作用标记 |
| `license` | 是 | 许可证 |
| `repository` | 否 | 仓库地址 |
| `keywords` | 否 | 搜索关键词 |
| `author` | 否 | 作者信息 |
| `contributors` | 否 | 贡献者列表 |
| `engines` | 否 | 运行环境版本要求 |

## 变体一：零依赖模块

适用于不依赖任何外部运行时库的独立工具模块。


## 变体二：有依赖模块

适用于依赖仓库内其他模块或外部库的模块。

```json
{
  "name": "@｛｛作用域｝｝/｛｛模块名称｝｝",
  "version": "0.1.0",
  "description": "｛｛描述｝｝",
  "main": "src/index",
  "module": "src/index",
  "types": "src/index.d",
  "exports": {
    ".": {
      "types": "src/index.d",
      "import": "src/index.mjs",
      "require": "src/index.cjs"
    }
  },
  "files": [
    "src",
    "README.md",
    "CHANGELOG.md",
    "LICENSE"
  ],
  "sideEffects": false,
  "scripts": {
    "build": "｛｛构建命令｝｝",
    "test": "｛｛测试命令｝｝",
    "lint": "｛｛代码检查命令｝｝",
    "typecheck": "｛｛类型检查命令｝｝"
  },
  "dependencies": {
    "@｛｛作用域｝｝/｛｛内部依赖｝｝": "｛｛工作区协议｝｝:｛｛工作区路径｝｝",
    "｛｛外部依赖｝｝": "｛｛外部依赖版本｝｝"
  },
  "devDependencies": {
    "｛｛构建工具｝｝": "｛｛构建工具版本｝｝",
    "｛｛测试框架｝｝": "｛｛测试框架版本｝｝",
    "｛｛代码检查工具｝｝": "｛｛代码检查工具版本｝｝",
    "｛｛类型检查工具｝｝": "｛｛类型检查工具版本｝｝"
  },
  "peerDependencies": {
    "｛｛同行依赖｝｝": "｛｛同行依赖版本范围｝｝"
  },
  "license": "｛｛许可证｝｝",
  "repository": {
    "type": "｛｛仓库类型｝｝",
    "url": "｛｛仓库地址｝｝"
  },
  "keywords": ["｛｛关键词1｝｝", "｛｛关键词2｝｝"],
  "author": "｛｛作者｝｝"
}
```

**依赖说明**：

| 依赖类型 | 字段 | 说明 |
|----------|------|------|
| 内部依赖 | `dependencies` | 使用工作区协议引用仓库内其他模块 |
| 外部依赖 | `dependencies` | 使用版本范围引用外部库 |
| 对等依赖 | `peerDependencies` | 由宿主环境/使用者提供的依赖 |

## 变体三：CLI 工具模块

```json

  },
  "files": [
    "src",
    "README.md",
    "CHANGELOG.md",
    "LICENSE"
  },
  "scripts": {
    "build": "｛｛构建命令｝｝",
    "test": "｛｛测试命令｝｝",
    "lint": "｛｛代码检查命令｝｝"
  },
  "dependencies": {
    "｛｛CLI框架｝｝": "｛｛CLI框架版本｝｝"
  },
  "devDependencies": {
    "｛｛构建工具｝｝": "｛｛构建工具版本｝｝",
    "｛｛测试框架｝｝": "｛｛测试框架版本｝｝",
    "｛｛代码检查工具｝｝": "｛｛代码检查工具版本｝｝"

    "type": "｛｛仓库类型｝｝",
```

## 变体四：聚合模块（Barrel / Re-export）

适用于将多个子模块统一导出的聚合模块，本身不包含实现代码。

```json
{
  "name": "@｛｛作用域｝｝/｛｛模块名称｝｝",
  "version": "0.1.0",
  "description": "｛｛描述｝｝ — 聚合导出模块",
  "main": "src/index",
  "module": "src/index",
  "types": "src/index.d",
  "exports": {
    ".": {
      "types": "src/index.d",
      "import": "src/index.mjs",
      "require": "src/index.cjs"
    }
  },
  "files": [
    "src",
    "README.md",
    "CHANGELOG.md",
    "LICENSE"
  ],
  "sideEffects": false,
  "scripts": {
    "build": "｛｛构建命令｝｝",
    "lint": "｛｛代码检查命令｝｝"
  },
  "dependencies": {
    "@｛｛作用域｝｝/｛｛子模块A｝｝": "｛｛工作区协议｝｝:｛｛子模块A路径｝｝",
    "@｛｛作用域｝｝/｛｛子模块B｝｝": "｛｛工作区协议｝｝:｛｛子模块B路径｝｝",
    "@｛｛作用域｝｝/｛｛子模块C｝｝": "｛｛工作区协议｝｝:｛｛子模块C路径｝｝"
  },
  "devDependencies": {
    "｛｛构建工具｝｝": "｛｛构建工具版本｝｝",
    "｛｛代码检查工具｝｝": "｛｛代码检查工具版本｝｝"
  },
  "license": "｛｛许可证｝｝",
  "repository": {
    "type": "｛｛仓库类型｝｝",
    "url": "｛｛仓库地址｝｝"
  },
  "keywords": ["｛｛关键词1｝｝", "｛｛关键词2｝｝"],
  "author": "｛｛作者｝｝"
}
```

**聚合模块特点**：

- 源码仅包含 re-export 语句，无业务逻辑
- 不需要测试文件（无自身逻辑可测）
- 依赖列表仅包含被聚合的子模块
- `sideEffects` 设为 `false` 以支持 tree-shaking
