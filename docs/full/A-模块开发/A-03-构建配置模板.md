# A-F03 构建配置模板（完整版）

> **模板 ID**: A-F03
> **模板名称**: 构建配置模板（完整版）
> **版本**: 1.0.0
> **适用范围**: 模块/包的完整构建配置，覆盖多种模块类型

---

## 模板说明

构建配置文件定义了模块从源代码到可分发产物的完整编译流程。
不同构建工具的配置格式各异，但核心概念一致。本模板使用通用的 JSON-like 格式，
可映射到任意构建工具。

---

## 通用字段参考

| 字段 | 必填 | 说明 |
|------|------|------|
| `entry` | 是 | 入口文件映射 |
| `output.formats` | 是 | 输出模块格式 |
| `output.dir` | 是 | 产物输出目录 |
| `output.fileName` | 否 | 自定义产物文件名 |
| `declaration` | 否 | 是否生成类型声明文件 |
| `declarationDir` | 否 | 声明文件输出目录 |
| `sourcemap` | 否 | 源码映射 |
| `target` | 否 | 编译目标环境 |
| `minify` | 否 | 是否压缩产物 |
| `external` | 否 | 外部依赖列表（不打包） |
| `treeshake` | 否 | Tree-shaking 配置 |
| `clean` | 否 | 构建前清理 |
| `define` | 否 | 编译时全局变量替换 |
| `alias` | 否 | 路径别名 |

## 变体一：零依赖模块

适用于不依赖外部库的独立工具模块，构建配置最简洁。

```json
{
  "entry": {
    "index": "src/index"
  },
  "output": {
    "formats": ["esm", "cjs"],
    "dir": "dist",
    "preserveModules": false
  },
  "declaration": true,
  "declarationDir": "dist",
  "sourcemap": true,
  "target": "｛｛目标环境｝｝",
  "minify": false,
  "treeshake": true,
  "clean": true
}
```

**产物结构**：

```
dist/
├── index.mjs          # ESM 格式
├── index.cjs          # CJS 格式
├── index.d.ts         # 类型声明
└── index.mjs.map      # 源码映射
```

---

## 变体二：有依赖模块

适用于依赖仓库内其他模块或外部库的模块，需配置外部依赖排除。

```json
{
  "entry": {
    "index": "src/index"
  },
  "output": {
    "formats": ["esm", "cjs"],
    "dir": "dist",
    "preserveModules": false
  },
  "declaration": true,
  "declarationDir": "dist",
  "sourcemap": true,
  "target": "｛｛目标环境｝｝",
  "minify": false,
  "external": [
    "@｛｛作用域｝｝/*",
    "｛｛外部依赖A｝｝",
    "｛｛外部依赖B｝｝"
  ],
  "treeshake": {
    "moduleSideEffects": false
  },
  "alias": {
    "@｛｛作用域｝｝/shared": "｛｛共享模块工作区路径｝｝"
  },
  "clean": true
}
```

**关键配置说明**：

| 配置项 | 说明 |
|--------|------|
| `external` | 使用通配符 `@scope/*` 排除所有工作区内依赖；逐一列出外部依赖 |
| `treeshake.moduleSideEffects` | 设为 `false` 表示依赖模块无副作用，允许更激进的 tree-shaking |
| `alias` | 将路径别名映射到工作区内实际路径，确保构建时正确解析 |

## 变体三：CLI 工具模块

适用于命令行工具模块，需输出可执行文件格式。

```json
{
  "entry": {
    "cli": "src/cli-entry",
    "index": "src/index"
  },
  "output": {
    "formats": ["esm", "cjs"],
    "dir": "dist",
    "binEntry": "cli",
    "binFormat": "executable"
  },
  "declaration": true,
  "declarationDir": "dist",
  "sourcemap": true,
  "target": "｛｛目标运行时｝｝",
  "minify": true,
  "external": [
    "｛｛外部依赖A｝｝",
    "｛｛外部依赖B｝｝"
  ],
  "treeshake": true,
  "define": {
    "｛｛包名称｝｝_VERSION": "｛｛清单文件版本｝｝"
  },
  "clean": true,
  "banner": {
    "js": "#!/usr/bin/env ｛｛运行时｝｝\n"
  }
}
```

**CLI 特有配置说明**：

| 配置项 | 说明 |
|--------|------|
| `binEntry` | 指定哪个入口作为可执行文件 |
| `binFormat` | 输出为平台原生可执行格式 |
| `define` | 编译时注入版本号等常量 |
| `banner` | 在产物头部添加 shebang 行（`#!/usr/bin/env`） |
| `minify` | CLI 工具建议开启压缩以减小体积 |

**产物结构**：

```
dist/
├── cli                # 可执行文件（平台原生格式）
├── index.mjs          # 库的 ESM 格式
├── index.cjs          # 库的 CJS 格式
├── index.d.ts         # 类型声明
└── cli.map            # CLI 源码映射
```

## 构建配置最佳实践

| 实践 | 说明 |
|------|------|
| 始终启用 `clean` | 避免旧产物残留导致混淆 |
| 开发阶段禁用 `minify` | 保留可读性，便于调试 |
| 生产构建启用 `minify` | 减小产物体积 |
| 正确配置 `external` | 避免将依赖打包进产物 |
| 启用 `sourcemap` | 便于线上问题排查 |
| 启用 `treeshake` | 减少未使用代码的体积 |
| 使用 `alias` 解析工作区路径 | 确保构建时路径解析正确 |
| 声明文件与产物同目录 | 简化使用者导入路径 |
