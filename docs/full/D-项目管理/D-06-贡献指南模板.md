## 3. 开发流程

### 3.1 分支管理

| 分支类型 | 命名格式 | 说明 |
|----------|----------|------|
| 功能开发 | `feature/｛｛任务编号｝｝-｛｛简述｝｝` | 新功能开发 |
| Bug修复 | `fix/｛｛任务编号｝｝-｛｛简述｝｝` | 问题修复 |
| 重构 | `refactor/｛｛简述｝｝` | 代码重构 |

### 3.2 开发步骤

1. **创建分支：** 从 `main` 创建功能分支
2. **开发实现：** 编写代码和测试
3. **本地验证：** 运行测试和构建
4. **提交代码：** 遵循提交规范
5. **推送分支：** 推送到远程仓库
6. **创建PR：** 填写PR描述模板

### 3.3 提交规范

**提交信息格式：**

```
｛｛类型｝｝(｛｛影响范围｝｝): ｛｛简要描述｝｝

｛｛详细描述（可选）｝｝

｛｛关联Issue（可选）｝｝
```

**提交类型：**

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug修复 |
| `docs` | 文档更新 |
| `style` | 代码格式调整 |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具相关 |

## 8. 发布流程

### 8.1 版本号规范

遵循语义化版本：`主版本.次版本.补丁版本`

| 变更类型 | 版本号变化 |
|----------|------------|
| 破坏性变更 | 主版本 +1 |
| 新功能（向后兼容） | 次版本 +1 |
| Bug修复 | 补丁版本 +1 |

### 8.2 发布步骤

1. 确认所有计划内功能已完成
2. 更新版本号
3. 更新变更日志
4. 运行完整测试
5. 创建发布标签
6. 构建并发布
7. 验证发布产物
