# A-F05 测试文件模板（完整版）

> **模板 ID**: A-F05
> **模板名称**: 测试文件模板（完整版）
> **版本**: 1.0.0
> **适用范围**: 模块/包的完整测试文件编写，覆盖多种测试类别

---

## 模板说明

测试文件采用"分组-用例-断言"的三层结构。完整版模板覆盖 5 种测试类别，
确保模块的全面质量保障。

---

## 测试类别概览

| 类别 | 英文 | 说明 | 优先级 |
|------|------|------|--------|
| 基本功能测试 | Unit / Functional | 验证核心功能的正确性 | 高 |
| 错误处理测试 | Error Handling | 验证异常和边界错误的处理 | 高 |
| 边界情况测试 | Edge Cases | 验证极端输入和特殊情况 | 中 |
| 类型/接口安全测试 | Type / Interface Safety | 验证类型约束和接口契约 | 中 |
| 性能测试 | Performance | 验证性能指标在可接受范围内 | 低 |

## 编写测试的最佳实践

| 实践 | 说明 |
|------|------|
| 每个测试只验证一个行为 | 便于定位失败原因 |
| 测试描述使用"应..."句式 | 清晰表达预期行为 |
| 使用参数化测试覆盖多组输入 | 减少重复代码 |
| 测试之间相互独立 | 不依赖执行顺序 |
| 测试数据与逻辑分离 | 使用 fixtures 目录管理测试数据 |
| 错误信息验证具体化 | 验证错误类型和消息内容，而非仅验证"抛出错误" |
