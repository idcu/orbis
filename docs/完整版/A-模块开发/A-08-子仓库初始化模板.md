# A-F08 子仓库初始化模板（完整版）

> **模板 ID**: A-F08
> **模板名称**: 子仓库初始化模板（完整版）
> **版本**: 1.0.0
> **适用范围**: 在多包仓库中完整初始化一个新的子模块/子仓库

---

## 模板说明

在多包仓库（monorepo）中，每个子模块需要遵循统一的目录结构和配置规范。
完整版模板包含子仓库所需的全部文件和配置。

---

## 一、目录结构

```
｛｛MODULE_NAME｝｝/
├── src/
│   ├── index            # 模块入口文件
│   ├── types            # 类型/接口定义
│   └── utils            # 内部工具函数（如有）
├── tests/
│   ├── setup            # 测试初始化脚本
│   ├── teardown         # 测试清理脚本
│   └── index.test       # 测试文件
├── ｛｛MANIFEST_FILE｝｝    # 模块清单文件
├── ｛｛BUILD_CONFIG_FILE｝｝# 构建配置文件
├── ｛｛TEST_CONFIG_FILE｝｝ # 测试配置文件
├── README.md            # 模块说明文档
├── CHANGELOG.md         # 变更日志
├── ｛｛GITIGNORE_FILE｝｝   # 忽略文件配置
└── ｛｛LINT_CONFIG_FILE｝｝ # 代码检查配置（如需覆盖）
```

## 八、初始化步骤清单

### 步骤 1：创建目录结构

```bash
mkdir -p packages/｛｛MODULE_NAME｝｝/{src,tests}
```

### 步骤 2：创建基础文件

按照上述模板创建以下文件：
- `src/index` — 模块入口
- `src/types` — 类型定义
- `tests/index.test` — 测试文件
- `｛｛MANIFEST_FILE｝｝` — 模块清单
- `｛｛BUILD_CONFIG_FILE｝｝` — 构建配置
- `｛｛TEST_CONFIG_FILE｝｝` — 测试配置
- `README.md` — 说明文档
- `CHANGELOG.md` — 变更日志
- `.gitignore` — 忽略配置

### 步骤 3：注册到工作区

在仓库根目录的工作区配置中添加模块路径。

### 步骤 4：安装依赖

```bash
# 在仓库根目录执行
｛｛WORKSPACE_INSTALL_COMMAND｝｝
```

### 步骤 5：验证

```bash
# 验证模块被正确识别
｛｛WORKSPACE_LIST_COMMAND｝｝

# 验证构建
｛｛WORKSPACE_BUILD_COMMAND｝｝ --filter=｛｛MODULE_NAME｝｝

# 验证测试
｛｛WORKSPACE_TEST_COMMAND｝｝ --filter=｛｛MODULE_NAME｝｝

# 验证代码检查
｛｛WORKSPACE_LINT_COMMAND｝｝ --filter=｛｛MODULE_NAME｝｝
```

### 步骤 6：提交

```bash
git add packages/｛｛MODULE_NAME｝｝
git commit -m "feat: 初始化 ｛｛MODULE_NAME｝｝ 模块"
```
