# B-F08 工作区配置模板（完整版）

> **模板 ID**: B-F08
> **模板名称**: 工作区配置模板（完整版）
> **版本**: 1.0.0
> **适用场景**: 完整工作区配置，支持嵌套工作区、目录协议、版本对齐及包过滤

---

## 概述

本模板提供一个完整工作区配置，支持：

- 多级工作区与嵌套工作区
- 目录协议（catalog protocol）实现版本对齐
- 包含/排除模式精确控制工作区范围
- 工作区包间的依赖管理规范

---

## 工作区配置

```yaml
# ｛｛项目名称｝｝ 工作区配置（完整版）
# 适用于包管理器（如 pnpm / npm / yarn workspaces / Cargo workspace 等）

# ====== 工作区定义 ======
workspace:
  packages:
    # --- 应用层 ---
    - "apps/*"
    - "apps/*/packages/*"           # 应用内的私有包

    # --- 库层 ---
    - "packages/*"
    - "packages/*/packages/*"       # 库内的私有包

    # --- 工具层 ---
    - "tools/*"

    # --- 模板层 ---
    - "templates/*"

  # --- 排除规则 ---
  exclude:
    - "**/test/**"                  # 排除测试目录
    - "**/examples/**"              # 排除示例目录
    - "**/dist/**"                  # 排除构建产物
    - "packages/deprecated/**"      # 排除已废弃的包

# ====== 目录协议（版本对齐） ======
catalog:
  # --- 运行时依赖 ---
  default:
    "｛｛核心依赖1｝｝": "｛｛核心依赖1版本｝｝"
    "｛｛核心依赖2｝｝": "｛｛核心依赖2版本｝｝"
    "｛｛工具库1｝｝": "｛｛工具库1版本｝｝"

  # --- 开发依赖 ---
  dev:
    "｛｛构建工具｝｝": "｛｛构建工具版本｝｝"
    "｛｛测试框架｝｝": "｛｛测试框架版本｝｝"
    "｛｛代码检查工具｝｝": "｛｛代码检查工具版本｝｝"
    "｛｛格式化工具｝｝": "｛｛格式化工具版本｝｝"
    "｛｛类型检查工具｝｝": "｛｛类型检查工具版本｝｝"

  # --- 同行评审依赖 ---
  peer:
    "｛｛运行时｝｝": "｛｛运行时版本范围｝｝"

# ====== 工作区行为 ======
settings:
  # --- 链接行为 ---
  linkWorkspacePackages: true       # 工作区包间自动链接
  preferWorkspacePackages: true     # 优先使用工作区包

  # --- 构建行为 ---
  autoInstallPeers: true            # 自动安装同行依赖
  strictPeerDependencies: false     # 严格同行依赖检查

  # --- 保存行为 ---
  savePrefix: ""                    # 保存版本时不加 ^ 前缀
  exact: true                       # 使用精确版本号
```

## 目录协议使用示例

工作区内的包通过 `catalog:` 协议引用目录中定义的版本：

```json
{
  "name": "@｛｛组织名｝｝/core",
  "dependencies": {
    "｛｛核心依赖1｝｝": "catalog:default",
    "｛｛核心依赖2｝｝": "catalog:default"
  },
  "devDependencies": {
    "｛｛构建工具｝｝": "catalog:dev",
    "｛｛测试框架｝｝": "catalog:dev",
    "｛｛代码检查工具｝｝": "catalog:dev"
  },
  "peerDependencies": {
    "｛｛运行时｝｝": "catalog:peer"
  }
}
```

## 目录协议说明

| 协议 | 说明 | 示例 |
|------|------|------|
| `catalog:default` | 引用 `catalog.default` 中定义的版本 | `"framework-core": "catalog:default"` |
| `catalog:dev` | 引用 `catalog.dev` 中定义的版本 | `"eslint": "catalog:dev"` |
| `catalog:peer` | 引用 `catalog.peer` 中定义的版本 | `"node": "catalog:peer"` |
| `workspace:*` | 引用工作区内的包（任意版本） | `"@myorg/core": "workspace:*"` |
| `workspace:^` | 引用工作区内的包（兼容版本） | `"@myorg/core": "workspace:^"` |
| `workspace:~` | 引用工作区内的包（近似版本） | `"@myorg/core": "workspace:~"` |

## 适配指南

- **包管理器 A (如 pnpm)**: 使用 `pnpm-workspace.yaml`，原生支持 `catalog:` 协议。
- **包管理器 B (如 npm)**: 在根 `package.json` 中定义 `workspaces`，目录协议可通过工具实现。
- **包管理器 C (如 yarn)**: 在根 `package.json` 中定义 `workspaces`，使用 `yarn workspace` 命令。
- **其他语言**: 参照各语言的包管理工作区机制。

---

## 注意事项

1. 嵌套工作区会增加配置复杂度，建议仅在必要时使用。
2. 目录协议有助于统一管理依赖版本，避免版本不一致问题。
3. 排除规则应明确，避免将构建产物或测试目录误识别为工作区包。
4. `preferWorkspacePackages: true` 可确保包管理器优先使用工作区内的包而非远程注册表的版本。
5. 建议在 CI 中定期检查依赖版本一致性。
