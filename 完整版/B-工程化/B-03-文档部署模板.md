# B-F03 文档部署模板（完整版）

> **模板 ID**: B-F03
> **模板名称**: 文档部署模板（完整版）
> **版本**: 1.0.0
> **适用场景**: 完整文档部署流程，包含构建、部署、PR 预览及多版本支持

---

## 概述

本模板提供一个完整文档部署流水线，包含 **3 个作业**：

1. **文档构建** - 构建文档站点，支持多版本
2. **文档部署** - 部署到静态托管/CDN
3. **文档预览** - 为 PR 生成预览链接并评论到 PR

---

## 流水线配置

```yaml
# {{项目名称}} 文档部署流水线（完整版）
# 适用于 CI 平台

# ====== 全局变量 ======
variables:
  PROJECT_NAME: "{{项目名称}}"
  DOCS_OUTPUT_DIR: "{{文档输出目录}}"
  DOCS_BASE_URL: "{{文档站点地址}}"
  PREVIEW_BASE_URL: "{{预览站点地址}}"

# ====== 并发控制 ======
concurrency:
  group: "docs-${{分支名称}}"
  cancel-in-progress: true

# ====== 触发条件 ======
triggers:
  branches:
    - "{{默认分支}}"
    - "{{文档分支}}"
  pull_requests:
    branches:
      - "{{默认分支}}"

# ====== 作业 1: 文档构建 ======
jobs:
  docs-build:
    name: "文档构建"
    runs-on: "{{运行环境}}"
    steps:
      - name: 检出代码
        uses: "checkout@v4"
        with:
          fetch-depth: 0  # 获取完整历史以支持多版本

      - name: 安装依赖
        run: "{{安装文档依赖命令}}"

      - name: 构建当前版本文档
        run: "{{构建文档命令}}"

      - name: 构建多版本文档
        run: |
          # 获取所有已发布的版本标签
          VERSIONS=$({{获取版本标签命令}})
          for VERSION in $VERSIONS; do
            echo "构建版本: $VERSION"
            {{切换到版本命令}} "$VERSION"
            {{构建指定版本文档命令}} --base "/$VERSION/"
            {{切换回主分支命令}}
          done

      - name: 构建版本索引页
        run: |
          # 生成版本选择页面
          {{生成版本索引命令}} --versions "$VERSIONS" --latest "{{最新版本}}"

      - name: 上传文档产物
        uses: "upload-artifact@v4"
        with:
          name: "docs-artifacts"
          path: "{{文档输出目录}}"

  # ====== 作业 2: 文档部署 ======
  docs-deploy:
    name: "文档部署"
    runs-on: "{{运行环境}}"
    needs: [docs-build]
    if: "${{分支名称 == 默认分支}}"
    steps:
      - name: 下载文档产物
        uses: "download-artifact@v4"
        with:
          name: "docs-artifacts"

      - name: 部署到 CDN
        uses: "deploy-cdn@v4"
        with:
          source-dir: "{{文档输出目录}}"
          destination: "{{CDN目标路径}}"
          cdn-provider: "{{CDN服务商}}"
        env:
          CDN_TOKEN: "${{CDN令牌}}"
          CDN_ENDPOINT: "${{CDN端点}}"

      - name: 配置 CDN 缓存策略
        run: |
          # 设置合理的缓存头
          {{配置缓存策略命令}} \
            --path "{{CDN目标路径}}" \
            --max-age "{{缓存最大时间}}" \
            --s-maxage "{{CDN缓存时间}}"

      - name: 验证部署
        run: |
          # 检查文档站点是否可访问
          STATUS=$(curl -s -o /dev/null -w "%{http_code}" "{{文档站点地址}}")
          if [ "$STATUS" != "200" ]; then
            echo "错误: 文档站点返回状态码 $STATUS"
            exit 1
          fi
          echo "文档部署成功: {{文档站点地址}}"

  # ====== 作业 3: 文档预览（PR） ======
  docs-preview:
    name: "文档预览"
    runs-on: "{{运行环境}}"
    needs: [docs-build]
    if: "${{事件类型 == pull_request}}"
    steps:
      - name: 下载文档产物
        uses: "download-artifact@v4"
        with:
          name: "docs-artifacts"

      - name: 部署预览
        uses: "deploy-preview@v4"
        with:
          source-dir: "{{文档输出目录}}"
          preview-id: "pr-${{PR编号}}"
        env:
          PREVIEW_TOKEN: "${{预览令牌}}"

      - name: 获取预览 URL
        id: preview
        run: |
          PREVIEW_URL="${{预览站点地址}}/pr-${{PR编号}}/"
          echo "url=$PREVIEW_URL" >> $GITHUB_OUTPUT

      - name: 评论预览链接到 PR
        uses: "comment-pr@v4"
        with:
          pr-number: "${{PR编号}}"
          body: |
            ## 文档预览已生成

            本次 PR 的文档预览已部署，可在以下地址查看：

            **预览地址**: ${{步骤输出.预览URL}}

            > 预览链接将在 PR 合并后自动失效。
```

## 多版本文档策略

```
文档站点根目录
├── index.html          (版本选择页)
├── v1.0.0/             (历史版本)
│   ├── index.html
│   └── ...
├── v1.1.0/
│   ├── index.html
│   └── ...
└── v2.0.0/             (最新版本)
    ├── index.html
    └── ...
```

- 每个版本独立目录，通过 URL 路径区分。
- 根目录提供版本选择页面，默认跳转到最新版本。
- 历史版本为只读，不再更新。

## 适配指南

- **GitHub Actions**: 使用 `actions/deploy-pages` 部署，`actions/github-script` 评论 PR。
- **GitLab CI**: 使用 `pages` job 类型，`artifacts:paths` 指定输出目录。
- **Jenkins**: 使用 `publishHTML` 或 `rsync` 部署，`httpRequest` 评论 PR。
- **其他平台**: 参照各平台文档进行语法映射。

---

## 注意事项

1. 多版本文档构建会显著增加构建时间，建议仅保留最近 N 个版本。
2. PR 预览部署应设置自动清理策略，避免预览资源累积。
3. CDN 缓存策略应区分 HTML（短缓存）和静态资源（长缓存 + 内容哈希）。
4. 预览链接应在 PR 合并后自动失效或定期清理。
