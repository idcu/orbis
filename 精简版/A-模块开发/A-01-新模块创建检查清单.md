## 检查清单

在创建新模块时，逐项确认以下内容：

### 1. 模块基本信息

- [ ] **模块名称**：`{{MODULE_NAME}}`（命名规范：小写、连字符分隔，与仓库命名一致）
- [ ] **版本号**：`0.1.0`（初始开发阶段）
- [ ] **描述**：一句话说明模块用途

### 2. 入口与导出

- [ ] **入口文件**：`{{ENTRY_FILE}}`（如 `src/index`）
- [ ] **导出清单**：确认所有公开接口已从入口文件导出

### 3. 脚本命令

- [ ] **构建命令**：`build` — 编译/打包模块
- [ ] **测试命令**：`test` — 运行测试
- [ ] **代码检查命令**：`lint` — 静态分析

### 4. 依赖声明

- [ ] **运行时依赖**：仅列出必要的运行时依赖
- [ ] **开发依赖**：构建工具、测试框架、代码检查工具等

### 5. 构建配置

- [ ] **输出格式**：ESM / CJS（根据目标环境选择）
- [ ] **声明文件**：启用类型声明文件生成（如适用）
- [ ] **产物清理**：构建前自动清除旧产物

### 6. 许可证

- [ ] **许可证类型**：与仓库主许可证保持一致
- [ ] **许可证字段**：在模块清单文件中声明

## 快速参考

| 检查项 | 必填 | 说明 |
|--------|------|------|
| 模块名称 | 是 | 全局唯一 |
| 版本号 | 是 | 遵循语义化版本 |
| 入口文件 | 是 | 模块的主入口 |
| 导出清单 | 是 | 公开 API 列表 |
| 脚本命令 | 是 | 至少包含 build、test、lint |
| 依赖声明 | 是 | 区分运行时/开发依赖 |
| 构建配置 | 是 | 输出格式与声明文件 |
| 许可证 | 是 | 与仓库一致 |
