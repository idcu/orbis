# 📌 精简版模板

> 精简版模板覆盖核心要素，适合快速起步、小型项目或内部工具开发。
> 每个模板均可一键切换到对应的完整版。

## 使用说明

1. **选择模板**：从左侧导航栏选择所需模板
2. **复制使用**：将模板内容复制到你的项目中
3. **替换占位符**：将 `&#123;&#123;PLACEHOLDER&#125;&#125;` 替换为实际内容
4. **按需扩展**：如需更完整的模板，点击右上角「切换到完整版」

## 模板分类

| 分类 | 模板数 | 说明 |
|:---:|:---:|:---|
| A — 模块开发 | 9 | 模块创建、清单文件、构建、测试、文档 |
| B — 工程化 | 9 | CI/CD、发布、Git 钩子、工作区配置 |
| C — 质量保障 | 7 | 代码评审、验收、PR 检查、E2E |
| D — 项目管理 | 6 | 需求、技术方案、API 文档、架构 |
| E — 开发任务 | 6 | 功能开发、Bug 修复、重构、优化 |

## 占位符说明

| 占位符 | 含义 | 示例 |
|:---|:---|:---|
| `&#123;&#123;MODULE_NAME&#125;&#125;` | 模块名称 | `utils`、`auth-service` |
| `&#123;&#123;VERSION&#125;&#125;` | 版本号 | `1.0.0` |
| `&#123;&#123;DESCRIPTION&#125;&#125;` | 描述 | `通用工具函数库` |
| `&#123;&#123;AUTHOR&#125;&#125;` | 作者 | `张三` |
| `&#123;&#123;LICENSE&#125;&#125;` | 许可证 | `MIT`、`Apache-2.0` |
| `&#123;&#123;REPOSITORY_URL&#125;&#125;` | 仓库地址 | `https://github.com/org/repo` |
| `&#123;&#123;SCOPE&#125;&#125;` | 组织/作用域 | `@myorg` |
| `&#123;&#123;ENTRY_POINT&#125;&#125;` | 入口文件 | `src/index.ts` |
| `&#123;&#123;EXPORT_NAME&#125;&#125;` | 导出名称 | `createApp`、`useRouter` |
