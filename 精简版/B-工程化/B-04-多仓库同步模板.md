# B-S04 多仓库同步模板（精简版）

> **模板 ID**: B-S04
> **模板名称**: 多仓库同步模板（精简版）
> **版本**: 1.0.0
> **适用场景**: 基础仓库镜像同步，将源仓库单向同步到目标仓库

---

## 概述

本模板提供一个最简仓库同步方案，将源仓库的代码单向镜像同步到目标仓库，适用于代码备份或跨平台同步场景。

---

## 同步配置

```yaml
# {{项目名称}} 多仓库同步（精简版）
# 适用于 CI 平台

# ====== 全局变量 ======
variables:
  SOURCE_REPO: "{{源仓库地址}}"
  TARGET_REPO: "{{目标仓库地址}}"

# ====== 触发条件 ======
triggers:
  schedule:
    - cron: "{{定时表达式}}"  # 定时同步
  manual: true  # 支持手动触发

# ====== 作业定义 ======
jobs:
  sync:
    name: "仓库同步"
    runs-on: "{{运行环境}}"
    steps:
      # --- 1. 克隆源仓库 ---
      - name: 克隆源仓库
        run: |
          git clone --mirror {{源仓库地址}} source-repo

      # --- 2. 推送到目标仓库 ---
      - name: 推送到目标仓库
        run: |
          cd source-repo
          git remote add target {{目标仓库地址}}
          git push target --mirror
```

---

## 占位符说明

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{{项目名称}}` | 项目名称 | `my-project` |
| `{{源仓库地址}}` | 源仓库的 Git 地址 | `https://github.com/org/repo.git` |
| `{{目标仓库地址}}` | 目标仓库的 Git 地址 | `https://gitlab.example.com/org/repo.git` |
| `{{定时表达式}}` | 定时同步的 Cron 表达式 | `0 */6 * * *`（每 6 小时） |
| `{{运行环境}}` | CI 执行环境 | `ubuntu-latest` |

---

## 适配指南

- **GitHub Actions**: 使用 `cron` 触发器或 `workflow_dispatch` 手动触发。
- **GitLab CI**: 使用 `schedules` API 或 `when: manual`。
- **Jenkins**: 使用 `pipelineTriggers` + `cron` 或手动构建按钮。
- **其他平台**: 参照各平台的定时任务和手动触发机制。

---

## 注意事项

1. `--mirror` 会同步所有分支和标签，包括已删除的引用。
2. 目标仓库的认证令牌应通过 CI 平台的安全变量注入。
3. 如需双向同步或冲突处理，请参考完整版模板（B-F04）。
