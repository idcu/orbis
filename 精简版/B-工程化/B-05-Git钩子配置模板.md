# B-S05 Git 钩子配置模板（精简版）

> **模板 ID**: B-S05
> **模板名称**: Git 钩子配置模板（精简版）
> **版本**: 1.0.0
> **适用场景**: 基础 Git 钩子，提交前执行代码检查和格式校验

---

## 概述

本模板提供一个最简 Git 钩子配置，在 `pre-commit` 阶段自动运行 **代码检查** 和 **格式校验**，防止不合规代码被提交。

---

## 钩子配置

```bash
#!/bin/sh
# {{项目名称}} pre-commit 钩子
# 文件路径: .git/hooks/pre-commit

echo "正在执行 pre-commit 检查..."

# --- 代码检查 ---
echo ">>> 代码检查中..."
{{代码检查命令}}
if [ $? -ne 0 ]; then
  echo "错误: 代码检查未通过，请修复后重新提交"
  exit 1
fi

# --- 格式校验 ---
echo ">>> 格式校验中..."
{{格式检查命令}}
if [ $? -ne 0 ]; then
  echo "错误: 代码格式不符合规范，请运行格式化命令后重新提交"
  exit 1
fi

echo "pre-commit 检查通过"
```

---

## 占位符说明

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{{项目名称}}` | 项目名称 | `my-project` |
| `{{代码检查命令}}` | 执行代码检查的命令 | `npm run lint` / `ruff check .` |
| `{{格式检查命令}}` | 执行格式校验的命令 | `npm run format:check` / `ruff format --check .` |

---

## 安装方式

将上述脚本保存为 `.git/hooks/pre-commit` 并赋予执行权限：

```bash
chmod +x .git/hooks/pre-commit
```

或使用 Git 钩子管理工具统一管理（推荐）：

```bash
# 使用 Git 钩子管理工具安装钩子
{{安装钩子命令}}
```

---

## 适配指南

- **Git 钩子管理工具**: 使用专用的 Git 钩子管理工具（如 Husky 或等效工具）统一管理钩子，确保团队成员共享相同的钩子配置。
- **直接安装**: 将钩子脚本放入 `.git/hooks/` 目录，适用于小型项目。
- **Makefile/脚本**: 在构建脚本中自动安装钩子。

---

## 注意事项

1. 直接放置在 `.git/hooks/` 的钩子不会被 Git 跟踪，建议使用 Git 钩子管理工具。
2. 如需暂存文件级别的检查（仅检查本次提交的文件），请参考完整版模板（B-F05）。
3. 钩子脚本应尽量快速执行，避免影响开发者体验。
