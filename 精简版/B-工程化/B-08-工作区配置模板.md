# B-S08 工作区配置模板（精简版）

> **模板 ID**: B-S08
> **模板名称**: 工作区配置模板（精简版）
> **版本**: 1.0.0
> **适用场景**: 基础工作区定义，将多个包组织在统一工作区中

---

## 概述

本模板提供一个最简工作区配置，定义工作区中包含的包及其位置。适用于任何支持工作区概念的包管理器。

---

## 工作区配置

```yaml
# {{项目名称}} 工作区配置
# 适用于包管理器（如 pnpm / npm / yarn workspaces / Cargo workspace 等）

# ====== 工作区定义 ======
workspace:
  packages:
    # 应用
    - "apps/*"
    # 库
    - "packages/*"
```

---

## 占位符说明

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{{项目名称}}` | 项目名称 | `my-project` |

```
│   ├── core/                 # 核心库
│   │   └── {{包清单文件}}
│   ├── utils/                # 工具库
│   │   └── {{包清单文件}}
│   └── types/                # 类型定义
│       └── {{包清单文件}}
├── {{工作区配置文件}}
└── {{包清单文件}}
```

---

## 包间引用

工作区内的包可以通过包名直接引用其他工作区包：

```json
{
  "dependencies": {
    "@{{组织名}}/core": "workspace:*",
    "@{{组织名}}/utils": "workspace:*"
  }
}
```

---

## 适配指南

- **包管理器 A (如 pnpm)**: 在 `pnpm-workspace.yaml` 中定义 `packages` 字段。
- **包管理器 B (如 npm)**: 在根 `package.json` 的 `workspaces` 字段中定义。
- **包管理器 C (如 yarn)**: 在根 `package.json` 的 `workspaces` 字段中定义。
- **其他语言 (如 Rust)**: 在 `Cargo.toml` 的 `[workspace] members` 中定义。
- **其他语言 (如 Go)**: 使用 `go.work` 文件定义模块。

---

## 注意事项

1. 工作区内的包名应使用统一的作用域前缀（如 `@{{组织名}}/`）。
2. `workspace:*` 协议表示使用工作区内的最新版本，发布时会自动替换为实际版本号。
3. 如需嵌套工作区、目录协议等高级功能，请参考完整版模板（B-F08）。
