# B-S09 根仓库配置模板（精简版）

> **模板 ID**: B-S09
> **模板名称**: 根仓库配置模板（精简版）
> **版本**: 1.0.0
> **适用场景**: 基础根仓库配置，包含常用脚本和基本元信息

---

## 概述

本模板提供一个最简根仓库配置，定义项目基本信息和常用脚本，作为工作区的统一入口。

---

## 根包清单

```json
{
  "name": "@{{组织名}}/{{项目名称}}",
  "version": "{{项目版本}}",
  "private": true,
  "description": "{{项目描述}}",
  "scripts": {
    "dev": "{{开发服务器命令}}",
    "build": "{{构建命令}}",
    "test": "{{测试命令}}",
    "lint": "{{代码检查命令}}",
    "clean": "{{清理命令}}"
  }
}
```

---

## 占位符说明

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{{组织名}}` | 组织/团队作用域 | `myorg` |
| `{{项目名称}}` | 项目名称 | `my-project` |
| `{{项目版本}}` | 项目版本号 | `0.0.0` |
| `{{项目描述}}` | 项目描述 | `My Project Monorepo` |
| `{{开发服务器命令}}` | 启动开发服务器 | `turbo run dev` |
| `{{构建命令}}` | 构建所有包 | `turbo run build` |
| `{{测试命令}}` | 运行所有测试 | `turbo run test` |
| `{{代码检查命令}}` | 代码检查 | `eslint .` |
| `{{清理命令}}` | 清理构建产物 | `rm -rf dist` |

---

## 使用方式

```bash
# 从根目录运行命令
{{包管理器命令}} run build    # 构建所有包
{{包管理器命令}} run test     # 运行所有测试
{{包管理器命令}} run lint     # 代码检查
{{包管理器命令}} run dev      # 启动开发服务器
```

---

## 适配指南

- 根包清单文件名根据包管理器不同而不同（如 `package.json`、`Cargo.toml` 等）。
- 脚本命令应使用包管理器支持的工作区过滤功能。
- 如需更完整的根仓库配置（开发依赖、引擎约束、编辑器配置等），请参考完整版模板（B-F09）。

---

## 注意事项

1. 根包清单应标记为 `private: true`，避免被误发布。
2. 版本号通常设为 `0.0.0`，因为根包本身不发布。
3. 脚本命令应简洁，具体逻辑委托给工作区中的各包或共享脚本。
