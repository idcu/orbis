# B-S03 文档部署模板（精简版）

> **模板 ID**: B-S03
> **模板名称**: 文档部署模板（精简版）
> **版本**: 1.0.0
> **适用场景**: 基础文档构建与部署到静态托管服务

---

## 概述

本模板提供一个最简文档部署流水线，当代码推送到主分支时，自动 **构建文档 → 部署到静态托管服务**。

---

## 流水线配置

```yaml
# {{项目名称}} 文档部署流水线（精简版）
# 适用于 CI 平台

# ====== 全局变量 ======
variables:
  PROJECT_NAME: "{{项目名称}}"
  DOCS_OUTPUT_DIR: "{{文档输出目录}}"

# ====== 触发条件 ======
triggers:
  branches:
    - "{{默认分支}}"

# ====== 作业定义 ======
jobs:
  docs:
    name: "构建并部署文档"
    runs-on: "{{运行环境}}"
    steps:
      # --- 1. 检出代码 ---
      - name: 检出代码
        uses: "checkout@v4"

      # --- 2. 安装依赖 ---
      - name: 安装依赖
        run: "{{安装文档依赖命令}}"

      # --- 3. 构建文档 ---
      - name: 构建文档
        run: "{{构建文档命令}}"

      # --- 4. 部署到静态托管 ---
      - name: 部署文档
        uses: "deploy-pages@v4"
        with:
          source-dir: "{{文档输出目录}}"
          destination-url: "{{文档站点地址}}"
```

---

## 占位符说明

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{{项目名称}}` | 项目或仓库的名称 | `my-project` |
| `{{文档输出目录}}` | 文档构建输出目录 | `docs/dist` |
| `{{默认分支}}` | 主分支名称 | `main` |
| `{{运行环境}}` | CI 执行环境 | `ubuntu-latest` |
| `{{安装文档依赖命令}}` | 安装文档构建依赖 | `npm ci` |
| `{{构建文档命令}}` | 构建文档站点的命令 | `npm run docs:build` |
| `{{文档站点地址}}` | 部署后的文档 URL | `https://docs.example.com` |

---

## 适配指南

- **GitHub Actions**: 使用 `actions/deploy-pages` 或第三方部署 action。
- **GitLab CI**: 使用 `pages` job 类型，`artifacts.paths` 指定输出目录。
- **Jenkins**: 使用 `publishHTML` 插件或 `rsync` 部署到目标服务器。
- **其他平台**: 使用平台提供的静态站点部署功能或 `scp`/`rsync` 部署。

---

## 注意事项

1. 确保文档构建命令输出到 `{{文档输出目录}}` 指定的路径。
2. 静态托管服务需提前配置好域名和访问权限。
3. 如需 PR 预览功能，请参考完整版模板（B-F03）。
