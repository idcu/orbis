# B-S02 发布流水线模板（精简版）

> **模板 ID**: B-S02
> **模板名称**: 发布流水线模板（精简版）
> **版本**: 1.0.0
> **适用场景**: 基础发布流程，主分支合并后自动构建、测试并发布到包注册表

---

## 概述

本模板提供一个最简发布流水线，当代码合并到主分支时，自动执行 **构建 → 测试 → 发布** 三步，将产物发布到包注册表。

---

## 流水线配置

```yaml
# {{项目名称}} 发布流水线（精简版）
# 适用于 CI 平台

# ====== 全局变量 ======
variables:
  PROJECT_NAME: "{{项目名称}}"
  REGISTRY_URL: "{{包注册表地址}}"

# ====== 触发条件 ======
triggers:
  branches:
    - "{{默认分支}}"

# ====== 作业定义 ======
jobs:
  publish:
    name: "构建 / 测试 / 发布"
    runs-on: "{{运行环境}}"
    steps:
      # --- 1. 检出代码 ---
      - name: 检出代码
        uses: "checkout@v4"

      # --- 2. 安装依赖 ---
      - name: 安装依赖
        run: "{{安装依赖命令}}"

      # --- 3. 运行测试 ---
      - name: 运行测试
        run: "{{运行测试命令}}"

      # --- 4. 构建 ---
      - name: 构建
        run: "{{构建命令}}"

      # --- 5. 发布到注册表 ---
      - name: 发布到 {{包注册表地址}}
        run: "{{发布命令}}"
        env:
          REGISTRY_TOKEN: "${{发布令牌}}"
```

---

## 占位符说明

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{{项目名称}}` | 项目或仓库的名称 | `my-lib` |
| `{{包注册表地址}}` | 包注册表的 URL | `registry.example.com` |
| `{{默认分支}}` | 主分支名称 | `main` |
| `{{运行环境}}` | CI 执行环境 | `ubuntu-latest` |
| `{{安装依赖命令}}` | 安装项目依赖的命令 | `npm ci` |
| `{{运行测试命令}}` | 执行测试的命令 | `npm test` |
| `{{构建命令}}` | 执行项目构建的命令 | `npm run build` |
| `{{发布命令}}` | 发布到注册表的命令 | `npm publish --registry=...` |
| `{{发布令牌}}` | 发布所需的认证令牌 | `${{secrets.NPM_TOKEN}}` |

---

## 适配指南

- **GitHub Actions**: 将 `env.REGISTRY_TOKEN` 映射为 `${{ secrets.XXX }}`。
- **GitLab CI**: 使用 `CI_JOB_TOKEN` 或 `variables` 中配置令牌。
- **Jenkins**: 使用 `withCredentials` 绑定令牌。
- **其他平台**: 参照各平台的安全变量/密钥管理方式。

---

## 注意事项

1. 发布令牌应通过 CI 平台的安全变量机制注入，切勿硬编码。
2. 建议在发布前增加版本号检查步骤，避免重复发布。
3. 如需更完整的发布流程（自动版本号、变更日志、发布说明），请参考完整版模板（B-F02）。
